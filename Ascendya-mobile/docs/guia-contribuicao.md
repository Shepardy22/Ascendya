# Guia de Contribuição — Ascendya

## Como contribuir
1. **Fork e clone o repositório.**
2. **Crie uma branch para sua feature ou correção:**
   ```bash
   git checkout -b minha-feature
   ```
3. **Siga o padrão de código:**
   - Use nomes descritivos para funções, variáveis e componentes.
   - Documente funções e componentes novos.
   - Mantenha a estrutura dos dados consistente.
4. **Adicione novos capítulos/tarefas em `public/data/` seguindo o modelo existente.**
5. **Teste sua alteração localmente antes de abrir um PR.**
6. **Abra um Pull Request detalhando sua contribuição.**

## Boas práticas
- Escreva commits claros e objetivos.
- Prefira componentes funcionais e hooks.
- Utilize o contexto global para dados do usuário.
- Documente mudanças relevantes na pasta `docs/`.

## Revisão de código
- Todos os PRs passam por revisão.
- Feedbacks e sugestões são bem-vindos.

## Contato
Dúvidas? Abra uma issue ou entre em contato pelo repositório.
